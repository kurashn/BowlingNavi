# ボウナビ AI記事執筆ガイドライン v3.0

AIが自動で記事を執筆し、WordPressに入稿する際は、必ず以下のガイドラインを**全項目厳守**してください。

---

## 1. 文字数・構成（必須要件）

- **総文字数**: 【絶対厳守】最低3,000文字以上。文字数が3,000文字を下回ることは絶対に許容されません。見出しごとに具体例や詳しい解説を豊富に盛り込み、最低3,000文字以上（目安は3,000〜5,000文字）で執筆すること。
- **必須の記事構成**:
  1. **リード文（ペルソナの悩みを明確化）**：最初の `<h2>` の前に、必ず**ターゲット読者（ペルソナ）の具体的な悩み**を明確にするリード文を配置する。抽象的な問いかけではなく、読者が「まさに自分のことだ」と感じるレベルで具体的に書くこと。
     **【リード文の構成（絶対厳守）】**
     `<p>「ボウリング場でマイボールを使っている人を見て、自分も欲しくなったけど何を選べばいいかわからない…」</p>`
     `<p>「ネットで調べても種類が多すぎて、結局どれがいいのか決められない」</p>`
     `<p>こんな経験、ありませんか？</p>`
     `<p>実は、初心者のマイボール選びで失敗する原因のほとんどは「重さ」と「素材」の2つに絞られます。</p>`
     `<p>本記事では、〇〇を△△の視点から詳しくお伝えします。</p>`
     ※悩みは**2〜3個を具体的なセリフ調**で列挙し、その後に「共感 → 原因の端的な提示 → 記事の内容紹介」の流れで書く。
  2. **本文**（H2ブロック × 4〜6個）
  3. **まとめ**：`<h2>まとめ</h2>` を配置し、記事の要点を箇条書きで振り返りつつ読者に次のアクションを促す
  4. **よくある質問（FAQ）**：必ず **【まとめの後に】** `<h2>よくある質問（FAQ）</h2>` を配置し、その下にQ&A形式で2〜3個の質問と回答を配置する
  ※ **目次（見出し一覧）は出力しないこと**（WordPressプラグイン側で自動生成するため）
- **基本セクション構成（絶対ルール）**:
  本文の各 `<h2>` の直下には、必ず「1〜2文程度の短い導入文（`<p>`タグ）」を配置し、そのあとに `<h3>` を複数（2〜3個）並べる基本構成を徹底してください。
  **【構成イメージ】**
  `<h2>大見出し</h2>`
  `<p>導入文（1〜2文で簡潔に）</p>`
  `<h3>小見出し1</h3>`
  `<p>短い段落（2〜3文）</p>`
  `<p>短い段落（2〜3文）</p>`
  `<h3>小見出し2</h3>`
  `<p>短い段落（2〜3文）</p>`

## 2. 出力HTMLのルール（絶対厳守）

- `class`・`id`・`style` 属性は**一切禁止**（特に `<div style="...">` のような直書きの背景ボックスは絶対に使用しないこと）
- `<span>` タグ・`<div>` タグは**原則禁止**
- 太字は `<strong>` ではなく **`<b>`** のみ使用
- 使用可能タグ: `<h2>` `<h3>` `<h4>` `<p>` `<ul>` `<ol>` `<li>` `<b>` `<mark>` `<a>` `<table>` `<tr>` `<th>` `<td>` `<blockquote>` `<hr>`
- **表や箇条書きの積極活用**: 長い文章が連続して読者が飽きないよう、要素の比較や情報の整理には**表（`<table>`）**を、手順やポイントの列挙には**箇条書き（`<ul>`, `<ol>`）**を積極的に使用すること。適度に視覚的な変化（装飾）を持たせることが必須です。
- **マーカー装飾**: 特に重要なキーワードや結論部分には、必ず **`<mark>重要なテキスト</mark>`** のようにマーカー装飾を使用すること。
- **装飾背景ボックス**: 枠で囲みたいような強調テキスト（旧来の div style で囲っていた部分）がある場合は、独自のスタイル付き `<div>` は使わず、後述するWordPress標準の装飾の背景ボックスのクラス（※現在クラス名確認中につき、一旦 `<blockquote>` で代用）を使用すること。

## 3. 文章品質の10原則（AI感の完全排除・読みやすさ最優先）

1. **結論を先出し（PREP法）**: 結論→理由→具体例→結論の順で書く
2. **文末表現（です・ます調）**: 全体を親しみやすい「です・ます調」で統一し、「だ・である」は使用しない。ただし「〜です。〜ます。」など同じ文末が3回以上連続しないよう、体言止めや疑問形（〜でしょうか？）、感嘆表現を織り交ぜて単調にならないリズムを作ること
3. **定型フレーズの禁止**: 「こんにちは」「この記事では〜」「いかがでしたか」「ぜひ参考にしてください」は一切使用しない
4. **曖昧表現の禁止**: 「〜と言われています」「一般的に〜」は使わず言い切る
5. **接続詞の連続禁止**: 「また」「しかし」「そして」「さらに」を連続で使わない
6. **見出しを具体的に**: 「メリット」ではなく「アベレージが10本上がった理由」のような具体的な見出しにする
7. **まとめの書き方**: 最後の「まとめ」セクションでは、記事の要点を**箇条書き（`<ul>`）で3〜5個**に整理し、その後に記事テーマに即した前向きな一言で締めくくる
8. **【最重要ルール】段落は短く、呼吸を持たせる**: スマートフォンでの可読性を最優先する。
   - 1つの `<p>` タグは **最大2〜3文（60〜80文字程度）** に収めること
   - **4行以上続く文字の塊は絶対に作らない**。こまめに `<p>` タグを分けて空白行（マージン）を作ること
   - 長い説明が必要な場合は、文章で続けるのではなく**箇条書き（`<ul>`）や表（`<table>`）に切り替える**こと
9. **【最重要ルール】視覚的な変化（リズム）を作る**: 読者は文章だけが延々と続くと離脱する。以下のルールで、視覚的に「飽きない」レイアウトを作ること。
   - **h3見出し1つにつき、最低1つ**は「表・箇条書き・`<blockquote>`・`<mark>`」のいずれかの装飾要素を含めること
   - テキストだけの `<p>` が **5個以上連続することは禁止**。途中で必ず装飾要素を挟むこと
   - 比較・スペック・選び方のポイントなどは、文章ではなく**表（`<table>`）**で整理すること
   - 重要な結論やポイントは **`<mark>`** や **`<blockquote>`** で視覚的に目立たせること
10. **情報の具体性**: 「おすすめです」だけで終わらず、「なぜおすすめなのか」を必ず1〜2文で補足する。価格・サイズ・重さなど、読者が購入判断に使える具体的な数値を必ず含める

## 4. ストーリー・体験の注入（最重要）

単なる情報の羅列ではなく、**読者が「自分ごと」として読める記事**を書く。

- **「あるある」から始める**: 「ガターばかりで恥ずかしい」「親指が抜けなくて困っている」など、読者が共感できるシーンから入る
- **失敗→原因→解決策の流れ**: 「なぜ上手くいかないのか」の原因を丁寧に解説し、解決策を提示する
- **具体的な数字を使う**: 「少し曲げる」ではなく「3〜5度傾ける」「ボール重量の10〜12%が目安」など
- **プロの目線を加える**: 「プロボウラーの山下選手が監修するボウナビでは〜」のような権威付けを適宜入れる

## 5. E-E-A-T（検索品質の最高基準）

| 要素 | 実装方法 |
|------|---------|
| **経験(Experience)** | 「実際にボウリング場で〜したとき」「練習で気づいた〜」のような一人称視点の体験エピソードを1記事につき最低1箇所入れる |
| **専門性(Expertise)** | ボウリングの技術用語（スパット・アプローチ・リリースポイント・オイルパターン等）を正確に使い、簡潔に解説する |
| **権威性(Authoritativeness)** | 「JPBA（日本プロボウリング協会）公認」「AMF公式ルール」など正式な団体名・基準を引用する |
| **信頼性(Trustworthiness)** | 数値や事実に基づく情報のみを記載。不明確な情報は「〜とされる（要確認）」と明記するか省く |

## 6. カテゴリー別の執筆戦略

### Gear（道具・用品）カテゴリー
- **目的**: アフィリエイト購買を促す
- **必須要素**:
  - 「こんな人におすすめ」ボックスを冒頭に配置
  - 比較する際は表（`<table>`）を使う
  - 価格帯の目安（〇〇円台）を必ず書く
  - 「初心者向け」「中級者向け」「本格競技向け」で分類する
  - 最後に「どこで買うか」の誘導（Amazonリンクなど想定）を書く

### Technique（テクニック）カテゴリー
- **目的**: 検索流入・SNSシェアを促す
- **必須要素**:
  - 「よくある失敗と原因」を必ず1セクション設ける
  - 「練習ドリル」や「チェックポイント」を箇条書きで提示
  - 上達の目安（「1ヶ月練習すると〇〇が変わる」等）を書く

### Rules & Knowledge / Knowledge カテゴリー
- **目的**: SEO流入・大会への送客
- **必須要素**:
  - 記事末尾に必ず「ボウナビで大会を探す」への自然な誘導文を1段落入れる
  - 例：「関西でボウリングの大会に出てみたいなら、ボウナビで条件別に検索してみよう。初心者歓迎の大会も多数掲載されている。」

### Health & Others カテゴリー
- **目的**: 幅広い読者の入口
- **必須要素**:
  - 医療・健康系の記事は「本記事は一般的な情報提供を目的としており、専門的な診断・治療の代わりにはなりません」を冒頭に明記
  - 楽しさや前向きな内容で締めくくる

## 7. 内部リンク戦略（SEO対策）

SEO効果を最大化するため、関連するテーマの記事へ積極的に内部リンク（アンカーテキスト）を設置してください。
- **参照元**: 執筆前に `content_pipeline.csv` を必ず確認し、関連する記事の `Slug` を探すこと。
- **リンク形式**: ドメインを含まない **ルート相対パス** の絶対厳守。
  - 〇 正: `<a href="/slug-name">自然な文脈のアンカーテキスト</a>`
  - × 誤: `<a href="https://www.bowlingnavi.com/slug-name">...</a>`
- **設置基準**: 
  1. 1記事につき **5〜10個** の内部リンクを設置すること。
  2. リンクの重複は避け、すべて異なる記事へのリンクとすること。
  3. **1つの段落に設置する内部リンクは最大1つまで**とすること。
  4. 「〇〇についてはこちら」のような単独の誘導文は避け、必ず**自然な日本語の文章の一部**としてリンク（アンカーテキスト）を組み込むこと。
  5. **導入文（最初のh2より前の文章）には内部リンクを一切設置しないこと**。（冒頭でのページ離脱を防ぐため、必ず本文のh2以降に配置する）

## 8. アフィリエイトリンク戦略（マネタイズ対策）

記事内で商品（マイボール、シューズ、バッグなど）をおすすめする場合、後から手動でアフィリエイトリンクを設置するための「プレースホルダー（目印）」を必ず出力してください。

- **プレースホルダーの書式**: `[アフィリエイトリンク挿入位置：検索推奨キーワード「〇〇」（Amazon/楽天/Yahoo）]`
- **設置基準**: 
  - 具体的な商品名を出しておすすめする直後や、比較表の下などに自然に設置する。
  - 1記事につき、必要に応じて1〜3箇所程度を上限とする。
  - AI自身が直接 `<a>` タグなどで適当なアフィリエイトURLを捏造することは**絶対に禁止**。必ず上記のプレースホルダー表記のみを出力すること。
  - 例: 「初心者にはABSのシューズがおすすめです。[アフィリエイトリンク挿入位置：検索推奨キーワード「ボウリングシューズ ABS S-380」（Amazon/楽天/Yahoo）]」

## 9. タイトル・SEOメタデータの自動生成ルール

記事を入稿する際、以下の**全フィールドを必ず設定**すること。

### 記事タイトル（`title` / H1）
- **メインキーワードを必ず含める**こと。
- キーワード間の**半角スペースは削除**し、自然な日本語の文章になるように繋ぐ。
- 例: `ボウリング マイボール 初心者セット` → `ボウリングのマイボール初心者セットおすすめ｜選び方のポイント`

### SEOタイトル（`fcb_seo_title`）
- 形式: `メインKW｜サブKW【2026年最新】` または `メインKW - サブ情報 | ボウナビ`
- 最大 **580px（約30〜32文字）** 以内
- 例: `ボウリングのマイボール初心者セットの選び方【2026年最新】`

### メタキーワード（`fcb_seo_keyword`）
- ロングテールKWをカンマ区切りで **3〜5個**
- 例: `ボウリング カーブ 投げ方, ボウリング カーブ コツ, ボウリング 曲がらない 原因`

### メタディスクリプション（`fcb_seo_description`）
- 最大 **990px（約60〜70文字）** 以内で結論を先出しした文章
- 「〜を解説します」で終わらず「〜がわかります」「〜を紹介する」で締める
- 例: `ボウリングのマイボール初心者セットに必要な用品の選び方と相場を徹底解説。2〜3万円で安く揃える方法がわかります。`

### パーマリンク（`slug`）
- CSVの `Slug` 列の値をそのまま使用する
- 例: `how-to-throw-curve-2026`

## 10. WordPressへの投稿APIコール（完全版）

### Step 1: 記事を下書き投稿（slug・SEOフィールドを同時に設定）

```
POST http://cms.bowlingnavi.com/wp-json/wp/v2/posts
Authorization: Basic <Base64(username:app_password)>
Content-Type: application/json

{
  "title":               "【記事タイトル（自然な日本語でキーワードを含む）】",
  "content":             "<h2>見出し</h2><p>本文...</p>",
  "status":              "draft",
  "slug":                "CSVのSlug列の値",
  "categories":          [カテゴリーのID（数値）],
  "fcb_seo_title":       "SEOタイトル（約32文字・580px以内）",
  "fcb_seo_keyword":     "KW1, KW2, KW3",
  "fcb_seo_description": "メタディスクリプション（約60〜70文字・990px以内）"
}
```

### Step 2: カテゴリーIDの取得（事前に確認）

```
GET http://cms.bowlingnavi.com/wp-json/wp/v2/categories?slug=gear
→ レスポンスの [0].id を使用
```

カテゴリースラッグとID対応表（初回確認後はキャッシュして使用）:
- `gear`      → 要確認
- `technique` → 要確認
- `knowledge` → 要確認
- `guide`     → 要確認
- `others`    → 要確認

### Step 3: 投稿完了後の処理

1. レスポンスから `id`（投稿ID）と `link`（WP管理画面URL）を取得
2. CSVの該当行の `Status` を `Draft` に更新
3. CSVの `WP URL` 列に管理画面URLを書き込む

### 完全な実行順序まとめ

1. CSVから記事を1件選択（gear → technique → knowledge 優先）
2. 記事を執筆（3000〜5000文字、ガイドライン全項目厳守）
3. 記事をWordPressに下書き投稿（`slug`・SEO全フィールド・カテゴリーID含む）
4. CSVのStatusを `Draft` に更新・WP URLを記録

※アイキャッチ画像（サムネイル）については手動でChatGPTにて生成・設定を行うため、AI側での画像生成・アップロード処理は不要です。
